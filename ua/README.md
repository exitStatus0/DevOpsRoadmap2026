# DevOps Roadmap 2026 — Повний курс українською

![DevOps Roadmap 2026 — Вступ](../en/Intro.png)

## Вікно можливостей у DevOps

Зараз — найкращий час увійти в DevOps. І ось чому.

Статистика вакансій DevOps 2025-2026

Попит на DevOps-інженерів **перевищує пропозицію** вже третій рік поспіль. Вакансії залишаються відкритими **значно довше**, ніж у класичній розробці. Компанії готові наймати junior-спеціалістів і навчати їх — якщо ті мають правильний фундамент.

Але є проблема: більшість початківців витрачають 12-18 місяців на **неправильні речі**. Збирають сертифікати замість практики. Вчать інструменти окремо, не розуміючи, як вони з'єднуються. Проходять десятки курсів, але не можуть розгорнути реальний застосунок.

Цей курс побудований навколо **5 факторів зростання** — саме тих напрямків, які визначають DevOps у 2026 році. Кожен модуль дає конкретні навички, які можна продемонструвати на співбесіді.

## 6 факторів зростання DevOps-інженера

| # | Фактор | Чому це критично | Модуль |
|---|--------|-------------------|--------|
| 1 | **Хмарне впровадження** (Cloud Adoption) | Більшість компаній використовують хмару; дефіцит cloud-інженерів зростає | [01-cloud-adoption](./01-cloud-adoption/) |
| 2 | **Контейнери та Kubernetes** | Kubernetes — стандарт оркестрації; Docker — мінімальний вхідний квиток | [02-containers-and-kubernetes](./02-containers-and-kubernetes/) |
| 3 | **DevSecOps** (Безпека) | Shift-left security — не опція, а вимога; витоки даних коштують мільйони | [03-devsecops](./03-devsecops/) |
| 4 | **Інфраструктура як код** (IaC) | Ручне налаштування — це технічний борг; Terraform / OpenTofu — мова інфраструктури | [04-infrastructure-as-code](./04-infrastructure-as-code/) |
| 5 | **ШІ та MLOps** | ШІ змінює щоденну роботу DevOps; MLOps — це на 80% DevOps-навички | [05-ai-and-mlops](./05-ai-and-mlops/) |
| 6 | **Спостережуваність та SRE** | Ви не можете полагодити те, чого не бачите. SLO, error budgets та distributed tracing — базові очікування | [06-observability-and-sre](./06-observability-and-sre/) |

## Структура кожного модуля

Кожен з 5 факторів має однакову структуру, щоб ви завжди знали, де шукати:

```
Фактор X: <Назва>
├── Чому це важливо у 2026
├── Яку проблему це вирішує в реальних командах
├── Що потрібно вивчити (ключові навички)
├── Що можна пропустити / «найгірший ROI»
├── Наскільки глибоко занурюватися (початківець → експерт)
├── Як ШІ змінює цей фактор
├── Типові помилки та пастки
├── Міні-практика (2–5 вправ)
├── «Сигнали» готовності до роботи
└── Посилання всередині репозиторію
```

## Як користуватися цим курсом

### Варіант 1: З нуля (немає досвіду в DevOps)

1. Почніть з [дорожньої карти](./90-roadmap/) — зрозумійте загальну картину
2. Прочитайте [типові помилки](./91-mistakes/) — щоб не повторювати їх
3. Пройдіть фактори **послідовно**: Cloud -> Контейнери -> IaC -> DevSecOps -> ШІ/MLOps
4. Виконуйте **кожну вправу** з розділу «Міні-практика»
5. Перевіряйте себе за чек-листами «Сигнали готовності»

**Орієнтовний час: 4-6 місяців** при навчанні 1-2 години на день.

### Варіант 2: З досвідом (є досвід у сисадмінстві, розробці або QA)

1. Перегляньте [дорожню карту](./90-roadmap/) — визначте, що вже знаєте
2. Для кожного фактора прочитайте розділ «Наскільки глибоко занурюватися» — знайдіть свій рівень
3. Зосередьтеся на **прогалинах**: якщо вмієте в Docker, але не в K8s — йдіть у [Фактор 2](./02-containers-and-kubernetes/)
4. Виконайте вправи рівня «впевнений» і вище

**Орієнтовний час: 2-3 місяці** для закриття прогалин.

### Варіант 3: Підготовка до співбесіди

1. Пройдіть **всі чек-листи** «Сигнали готовності» у кожному факторі
2. Те, що не можете пояснити — вивчіть глибше
3. Виконайте по одному портфоліо-проєкту з [дорожньої карти](./90-roadmap/)
4. Підготуйте відповіді на питання «Яку проблему це вирішує?» для кожного інструменту
5. Перечитайте [типові помилки](./91-mistakes/) — на співбесідах часто питають про anti-patterns

**Орієнтовний час: 2-4 тижні** інтенсивної підготовки.

## Принципи цього курсу

- **Практика > Теорія.** Кожен модуль завершується конкретними вправами
- **Глибина > Широта.** Краще знати один інструмент глибоко, ніж п'ять поверхнево
- **Розуміння > Запам'ятовування.** Ми пояснюємо «навіщо», а не тільки «як»
- **ШІ як помічник.** У кожному модулі показано, як ШІ прискорює роботу
- **Реальні сценарії.** Приклади з реальних команд, а не абстрактні діаграми

## Передумови

Перш ніж починати, переконайтеся, що у вас є:

- Базове розуміння Linux (командний рядок, файлова система, процеси)
- Будь-який досвід програмування (Python, Bash, або інша мова)
- Git на базовому рівні (clone, commit, push, branch)
- Англійська на рівні читання технічної документації

Якщо чогось бракує — розділ [Фаза 0 → 1](./90-roadmap/) у дорожній карті покриває ці основи.

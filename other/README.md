## Почему именно сейчас?

Три силы сошлись одновременно:

1. **Массовая миграция в облако** — даже консервативные компании (банки, госсектор, производство) переносят инфраструктуру в облако. Каждая миграция — это вакансия.
2. **ИИ как мультипликатор** — ИИ не заменяет DevOps-инженеров, а делает каждого инженера в 2-5 раз продуктивнее. Те, кто умеют использовать ИИ в инфраструктурных задачах, получают несправедливое преимущество.
3. **Безопасность стала обязательной** — DevSecOps перестал быть «бонусом» и стал обязательным требованием. Инженеры с пониманием безопасности стоят на 20-30% дороже.

## Структура курса

Каждый модуль-фактор содержит:

- **Почему это важно в 2026** — контекст рынка, а не абстрактная теория
- **Какую проблему это решает** — реальные кейсы из команд
- **Что нужно изучить** — конкретные навыки и инструменты
- **Что можно пропустить** — экономия времени, худший ROI
- **Уровни глубины** — новичок, уверенный, эксперт
- **Как ИИ меняет этот фактор** — практические примеры
- **Типичные ошибки** — чтобы не наступать на грабли
- **Мини-практика** — 2-5 упражнений для портфолио
- **Сигналы готовности** — чек-лист «я готов к работе»

## Как пользоваться этим курсом

### Вариант 1: Последовательное прохождение (рекомендуется для новичков)

1. Прочитай [Дорожную карту](./90-roadmap/) — пойми, на какой ты фазе
2. Прочитай [Типичные ошибки](./91-mistakes/) — чтобы не повторять чужие
3. Проходи факторы 1-5 последовательно, выполняя мини-практику
4. Собери портфолио из упражнений

### Вариант 2: Точечное изучение (для тех, кто уже в DevOps)

1. Оцени свои пробелы по таблице факторов
2. Погрузись в конкретный модуль
3. Выполни упражнения на уровне «уверенный» или «эксперт»

### Вариант 3: Подготовка к собеседованию

1. Пройди чек-листы «Сигналы готовности» по каждому фактору
2. Закрой пробелы через мини-практику
3. Используй портфолио-проекты из [Дорожной карты](./90-roadmap/)

## Принципы курса

1. **Практика > Теория.** Каждый модуль заканчивается конкретными упражнениями.
2. **Глубина > Ширина.** Лучше освоить одно облако на отлично, чем три поверхностно.
3. **ИИ — инструмент, а не замена.** Используй ИИ для ускорения, но понимай, что он генерирует.
4. **Безопасность с первого дня.** Не «потом добавим», а «встроено в процесс».
5. **Портфолио > Сертификаты.** Работающий проект на GitHub убедительнее любой бумажки.
